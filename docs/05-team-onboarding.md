# 05. 팀원 온보딩 + 피드백 수집

> Phase 1.5 | 예상: 1일 | 선행: 04-setup-scripts

## 목표

실제 팀원 2~3명에게 setup.sh를 돌려보게 하고, 문제점과 피드백을 수집한다. Phase 2 우선순위의 근거.

---

## 온보딩 시나리오 (10분)

```
0분   git clone + cd jetsong-toy
1분   ./setup.sh 실행
2분   [1/5] MCP 빌드 (자동)
3분   [2/5] Loki 공용 토큰 입력 (Slack 고정글에서 복사)
5분   [2/5] Jira 개인 토큰 발급 + 입력
7분   [2/5] GitLab 개인 토큰 발급 + 입력
8분   [5/5] 프로젝트 경로 입력 → Skill 심링크
9분   [4/5] Claude Code settings.json에 MCP 스니펫 붙여넣기
10분  도구 재시작 + 테스트
      → "내 Jira 티켓 보여줘"
      → "최근 API 에러 로그 확인해줘"
      → "내 MR 목록 보여줘"
```

## 피드백 수집 항목

| 항목 | 질문 |
|------|------|
| 시간 | 실제로 몇 분 걸렸나? |
| 막힌 곳 | 어디서 멈췄나? (토큰 발급? 경로 입력? 스니펫 붙여넣기?) |
| MCP | 3개 중 안 되는 거 있나? 어떤 에러? |
| Skill | /quick-review 실행해봤나? 유용했나? |
| 도구 | 주로 어떤 도구 쓰나? (Claude Code? Cursor?) |
| 희망사항 | 추가로 원하는 MCP나 Skill은? |
| 웹 UI | 쉘 대신 웹으로 세팅할 수 있으면 쓸 건가? |

## 피드백 수집 방식

Slack에 간단 스레드. 오버엔지니어링 금지:

```
@팀원 jetsong-toy 세팅 테스트 부탁!
1. git clone ... && ./setup.sh
2. 되면 👍, 막히면 어디서 막혔는지 답글
3. Claude Code에서 "내 Jira 티켓 보여줘" 해보고 결과 캡처
```

## 피드백 → Phase 2 우선순위 반영

| 피드백 패턴 | Phase 2 액션 |
|------------|-------------|
| "토큰 입력이 귀찮다" | `/setup` 웹 UI 우선 |
| "어떤 Skill이 있는지 모르겠다" | `/market` 리더보드 우선 |
| "설치 커맨드를 외울 수 없다" | CLI + 복사 버튼 우선 |
| "Slack MCP도 있으면 좋겠다" | MCP 추가 백로그 |
| "리뷰가 너무 빡빡하다/약하다" | Skill 체크리스트 튜닝 |
